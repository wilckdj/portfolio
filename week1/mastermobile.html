Master Mobile UX Notes

  • It makes sense to me that mobile first design should be used for websites. However from my experience, most companies don't implement it well. BYU-I tried to fix their website to be mobile-friendly and they made it more confusing.
  • Following the first video, I just realized that I didn't fully grasp "Mobile First", because I was still thinking of responsive retrofitting to make it mobile-friendly before publishing, but still starting with desktop.
  • Be There, Be Useful, Be Quick. - 0 steps is better than 5 steps.
  • Remember that mobile users have limited reach. Don't put things in difficult spots.
  • Pay attention to the speed of loading. Optimize the files. Chrome's developer window allows for network speed testing.
  • The videos talk about reaching buttons that users want to press, but buttons that shouldn't be accidentally clicked should also be moved away from the grip.
  • Make your buttons clear and look like buttons. Make things easier and simpler, not cooler.
  • Form design: Mobile users need larger fields, and the labels should be shown when field is selected.
  • When designing the form, remember that mobile forms have a keyboard that pops up from the bottom and will cover part of it.
  • Allow private information to be hidden, but automatically shown.
  • Don't overwhelm with mobile advertisements. Make them small and simple.
  • Use a lot of empathy with your design.
