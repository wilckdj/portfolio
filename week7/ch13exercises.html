  • Ajax allows web pages to communicate asynchronously with a server and dynamically update webpages without reloading.
  • Clients request resources from a server, servers send responses back to client.
  • Javascript is a client-side language, Ajax allows it to request resources from a server.
  • Ajax can be practice with a local development server or a website that uses Cross-origin resource sharing (CORS)
  • Asynchronous means that the program doesn't have to stop and wait for the response.
  • Ajax originally sent back XML documents, but JSON is most commonly used now.
  • Application programming interfaces (API) are methods that allow external access to a program or service.
  • The fetch API is the standard for requesting and sending data.
  • Fetch uses promises to avoid callback issues.
  • fetch() has only one mandatory argument, the URL being fetched.
  • Fetch uses promises, so the catch statement can deal with errors.
  • The response interface for fetch has an ‘ok’ property that is true if the response was successful/created.
  • The redirect() method can be used to redirect to another URL.
  • text() method takes a stream of text and returns a promise.
  • blob() reads a file of raw data, such as an image or a spreadsheet. The promise resolves with a blob object.
  • JSON is the most common format for AJAX responses. json() transforms a stream of JSON into a promise.
  • You can create your own response object using a constructor: const response = new Response(“”,{});
  • By providing a Request object you get more control over the request being made.
  • There are five commonly used HTTP verbs (methods): GET, POST, PUT, PATCH, DELETE
  • HTTP headers pass on additional information about a request or response.
  • AJAX can also be used to send information to a server.
  • AJAX is a living standard, meaning it is developed in the wild. It can and will change as feedback is provided about how it works.
  • If you don’t want to deal with the “wild”, libraries like jQuery can handle your AJAX requests.
