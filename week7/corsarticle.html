Week 7 Notes

CORS Article

  • Cross-Oriding Resource Sharing (CORS) allows a server to indicate any other origins that should be permitted by the browser to lad resources from.
  • Browsers restrict cross-origin HTTP requests initiated from scripts for security reasons.
  • Unless the proper CORS header is included in the request, only resources from the same origin as the application can be requested.
  • Modern browsers use CORS in APIs such as XMLHttpRequest or Fetch.
  • CORS standard works by adding new HTTP headers that let servers describe which origins are permitted.
  • Browsers ‘preflight’ the request by soliciting supported methods from the server, then with approval from the server, sends the actual requests.
  • CORS failures result in errors that are not available to JavaScript, but can be looked at in the console.
  • XMLHttpRequest can make cross-site requests in any supporting browser.
  • Simple requests don’t trigger a CORS preflight. They have specific conditions, including that no event listeners are registered on the object returned.
  • The server can indicate what domains the access of the resource is restricted to using “Access-Control-Allow-Origin: * (or https://example.example)”
  • Preflighted requests occur when the browser first sends a request using the OPTIONS method. This determines if the request is safe to send.
  • The request can include “Access-Control-Request-Method:” or “Access-Control-Request-Headers: “ to make sure that the server will accept those methods and headers specified.
  • Access-Control-Max-Age gives the value in seconds for how long the response to the preflight request can be cached without sending another preflight request.
  • Not all browsers currently support redirects after a preflighted request.
  • Credentialed requests can be made with CORS that are aware of HTTP cookies and HTTP Authentication information.
  • By default, credentials are not sent. A specific flag has to be set for credentials to be sent.
  • CORS-preflight requests must never include credentials.
  • The response to a preflight request must specify “Access-Control-Allow-Credentials: true” to indicate an actual request can include credentials.
