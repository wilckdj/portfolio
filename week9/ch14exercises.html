  • HTML5 is the latest version of HTML for creating webpages.
  • Many APIs were made available in HTML5 and beyond.
  • The HTML5 specification is separated into modules that allow different features to be implemented without having to wait for others to be completed.
  • The ‘data-‘ attribute is a way of embedding data in a web page using custom attributes that are ignored by the browser. They are private to a page.
  • They must contain only lowercase letters, numbers, hyphens, dots, colons or underscores.
  • each element has a dataset property that can be sued to access any ‘data-‘ attribute it contains.
  • ’data-powers’ is accessed by ‘.dataset.powers’. ‘data-max-length’ is accessed by ‘dataset.maxLength’.
  • you could also still use .getAttribute(‘data-powers’);
  • APIs help developers access and control new features that appear in the latest devices. It is best practice to use feature detection before using any of the API methods.
  • Web Storage API (localStorage) is similar to cookies but has differences.
  • Web Storage does not have an actual data limit, but most browsers limit it at 5GB per domain.
  • Stored data does not automatically expire as with cookies.
  • If web storage is supported, window will have localStorage and sessionStorage. Local Storage is more permenant, sessionStorage is only for the current session.
  • localStorage can trigger event listeners through the ‘storage’ event that can report the .key, .oldValue. newValue, and .storageArea.
  • The Geolocation API is used to obtain the geographical position of the device. It can find the user’s exact location then link to nearby places.
  • navigator.geolocation.getCurrentPosition(); can return your position object.
  • The position object has several properties:
    • position.coords.latitude (.longitude)
    • position.speed - ground speed of the device, meters per second.
    • position.altitude - estimate of the devices altitude in meters above the WGS84 ellipsoid.
    • position.heading – the direction the device is moving in. In degrees, clockwise from North.
    • position.timestamp – the time the position was recorded.
    • position.accuracy (position.altitudeAccuracy) – how accurate the returned values are.
  • navigator.geolocation.watchposition() will call a function every time the position of the device is updated.
  • navigator.geolocation.clearWatch() stops the callback from being called.
  • Web workers allow processes to be run in the background adding support for concurrency in JavaScript.
  • Workers are created using the Worker() constructor function.
  • Workers use the concept of messages to communicate back and forth between the main script and the worker script.
  • Messages are posed with worker.postMessage(“hello”); they can be listened for with event listeners for “message” events.
  • Workers can be terminated after the tasks are completed, using .terminate() from the main script or self.close() from the worker script.
  • Shared Web Workers allow lots of different scripts on the same domain to access the same worker object.
  • Service Workers API allows a worker script to run in the background with the added benefit of being able to intercept network requests.
  • Websocket is a protocol that allows two-way communication with a server – push messaging.
  • Websocket uses event listeners and responses from a server to work.
  • Notification API allows you to show messages using the system’s notifications. Such as a popup in the corner of the screen.
  • Before notifications can be sent, permission must be granted by the user. Notification.requestPermission() returns a promis with the permission property set either to granted or denied.
  • Multimedia (audio and video) is also available through HTML5.
  • The ‘audio’ and ‘video’ tags insert audio and video clips into a web page. The text between the tags will only display if the browser does not support them.
  • Both the audio and video elements have many properties and methods to control the playback, change volume, and otherwise affect the media.
