Hover elements

Automatic animation

  • The CSS3 transform property lets you translate, rotate, scale, and/or skew any element.
  • The value of the transform property is one or more transform functions applied in the order provided.
  • Translation functions allow you to move elements left, right, up, or down. These are similar to the behavior of position:relative.
  • Translate(x,y) moves objects left-right (x) and top-bottom (y)
  • translatex or translaty functions can be used to move on only one axis.
  • Transforms don’t work on inline elements.
  • scale(x,y) scales the element by defined factors horizontally then vertically. Can also specify scalex() or scaley()
  • Don’t declare multiple transforms, because the second overrides the first. Add additional transformations to the end.
  • rotate() rotates an element around the point of origin (default is center) by a specified angle.
  • skew(x,y) skews an element along the x and y axes.
  • These effects can be mimicked with other css properties.
  • Transitions allow simple animations to occur using only css.
  • Transitions use three steps: Declare original state, declare final state, include transition functions.
  • 'transition:' can be used as shorthand to declare all transition properties.
    • transition-property specifies the property that the transition is applied to.
    • transition-duration is how long the transition will take. Can be specified with seconds (s) or milliseconds (ms)
    • transition-timing-function specifies how the pace of the transition acts, whether it is constant, starts fast, or ends fast, etc.
    • transition-delay introduces a delay before the transition begins. Specified with s or ms.
  • multiple transitions can be called with the individual properties or the shorthand, separated by commas.
  • the all keyword can allow all properties to transition with the same rate, speed and delay.
  • To create a css animation, first create a named animation then attach it to an element.
  • @keyframes are used to define the start and end frames of an animation.
  • keyframes use two keyterms, from and to, which evaluate to 0% and 100%, for how far aong the animation each keyframe is located.
  • animations have the following properties:
    • animation-name: used to attach a keyframes animation to an element.
    • animation-duration: length of time in seconds or milliseconds for animation to go from 0% to 100%.
    • animation-timing-function: determines how the animation progresses over its duration. Can use steps()
    • animation-iteration-count: how many times the animation will play through, can use ‘infinite’
    • animation-direction: changes the playthrough direction using normal, reverse, alternate, etc.
    • animation-delay: delay the start of the animation.
    • animation-fill-mode: defines what happens before the first animation begins and after the last aniation ends.
    • animation-play-state: defines whether the animation is running or paused.
  • Shorthand uses ‘animation:’
