Scalable Vector Graphics (SVG)

  • The HTML Canvas API is used to draw images through JavaScript.
  • Canvas can improve the performance of websites by avoiding the need to download images off the network.
  • Canvas was first developed by Apple, so it is similar to Quarts 2D.
  • The canvas element is put straight into the HTML, with the size declared in HTML instead of CSS.
  • The canvas is selected and manipulated through JavaScript.
  • You set the strokeStyle and fillStyle attributes with the desired colors prior to actually drawing the image.
  • You can draw rectangles with fillRect() and strokeRect()
  • Shapes can be filled with patterns instead of solid colors, such as images or gradients.
  • Other shapes can be drawn by creating paths. Paths create a blueprint for your lines, arcs, and shapes.
  • You can save canvas drawings to a URL, which can allow them to be saved locally.
  • You can draw specific images to a canvas as well using drawImage()
  • Images in the canvas can be manipulated to do things such as alter the color style to black and white, or some other style.
  • Similarly, videos can be color-changed with canvas manipulation.
  • Canvas can also allow text to be displayed, such as for error handling.
  • Canvas doesn’t create a DOM node, so it is essentially invisible to tools like screen readers.
  • Scalable Vector Graphics (SVG) are images that preserve their quality even as they blow up or shrink down.
  • SVG can do many of the same things as canvas including drawing paths, shapes, text, gradients, and patterns.
  • Drawing a circle in SVG is much easier than in canvas. It uses the 'circle' tag.
  • Drawing a rectangle is done using the 'rect' tag.
  • Canvas allows for pixel manipulation but operates in immediate mode. Meaning, once a shape is created it cannot be accessed, it would need to be redrawn.
  • Canvas can allow the image to be saves as a PNG or JPEG file.
  • Canvas is faster than SVG.
  • SVG operates in retained mode, meaning that the structure of the image is preserved in the XML document.
  • SVG has a more complete set of tools, but SVG cannot manipulate pixels as canvas does.
  • Drag and Drop is an API that allows certain elements to be draggable and then to specify what happens when they are dropped onto other elements.
  • Drag and Drop is not supported by Android or IOS, therefore it is not useful for mobile applications.
  • Drag and Drop can be combined with the File API to allow files to be dragged from a desktop onto the website.
  • The draggable attribute is added to an element. Because it isn’t a Boolean attribute it must be explicitly set to true.
  • A dragstart event listener should be added to the draggable items.
  • both dragover and drop event listeners can be added to the elements that the draggable items would be dropped on.
  • Drag and drop would be great for a card game application.
