Chapter 12

Some examples are in the console and code comments.

  • Object-Oriented Programming (OOP) is often used to model representations of objects in the real world.
  • OOP has three main concepts: encapsulation, polymorphism, and inheritance.
  • Encapsulation: The inner workings are kept hidden inside the object and the essential functions are exposed to the end user.
  • Polymorphism: The same process can be used for different objects, or various objects can share methods.
  • Inheritance: Taking the features of one object then adding some new features.
  • Unlike Java and Ruby that are class-based languages, JavaScript is a prototype-based language.
  • Constructor Functions can be used when creating objects to define the properties and methods of an object.
  • With contructor functions, you can creat new instances using the 'new' operator.
  • You can create constructor functions or use the built-in constructor functions.
  • JavaScript ES6 introduces the class declaration which does the same thing as a constuctor function, but is similar to writing a class in other programming languages.
  • It is good practice to capitalize constructor function names.
  • ES6 class declarations are preferable to the constructor function because they are easier to read and is implicitly in strict mode.
  • All objects have a constructor property that returns the constructor function that created it.
  • If you don't know the name of an objects constructor, you can use the constructor property to instantiate a copy.
    • ex: const greenDice = new redDice.constructor(10);
  • The static keyword can be used to create a static method, which can only be called from the class directly and not from instances.
  • Javascript uses prototypal inheritance, every class has a prototype property shared by every instance.
  • If you want to add things to the class affter it is created, you can use the prototype property of the class.
  • Properties that are inherited from the prototype porperty are not the object's own property.
  • The prototype object is live, so other instances of a class inherit new properties and methods automatically.
  • You can overwrite properties and methods inherited from an object's prototype by assigning a new value to them.
  • Prototype should be used to define any properties that will remain the same for every instance of the class.
  • Object methods in JavaScript are public by default, meaning they can be queried and changed by assignment.
  • Using variable scope, some properties and methods can be kept private inside of a class declaration.
  • Because a prototype is just another object, it has its own prototype, which has a prototype, causing a chain.
  • Enumerable properties are properties that show up when a for-in loop is used.
  • A class can inherit from another class using the extends keyword in a class declaration.
  • Polymorphism means that different objects can have the same method used in different ways.
  • Adding methods to built-in objects is known as monkey-patching, and is mostly frowned upon.
  • An alternative to monkey-patching is using the extends keyword to build a subclass for your own methods.
