Chapter 15

Some examples are in the console and code comments.

  • A module is a self-contained piece of code providing functions and methods that can then be used in other files and other modules.
  • Keeping code organized improves code maintainability.
  • Code in a module should have a single purpose.
  • Good modules can be swapped out without affecting other parts of a project.
  • If a piece of code relies on another to run, it is said to be tightly couples.
  • JavaScript started supporting modules in ES6.
  • A module has its own global scope, so variables created in the top level of a module can only be access in that module.
  • A module file is a normal JavaScript file that uses the keyword export to specify what is made availble from the module.
  • You import functions or variables as follows: import {name} from './file.js';
  • Default exports refer to a single variable, function or class that can be imported without being explicitly named.
  • You cannot have more than one default export.
  • When importing a default export, no curly brackets are needed, and the alias does not have to match the name from the actual module.
